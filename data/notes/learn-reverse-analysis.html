给大家出一道思考题：

用 C 语言设计一个程序，验证输入的密码是否是 12345678，如果验证成功，就输出 success，如果验证失败，则输出 failed。

大部分新手小白估计会这么写：

#include <stdio.h>
#include <string.h>

int main() {
    char buf[10] = {0};
    scanf("%s", buf);

    if (strcmp(buf, "12345678") == 0) {
        printf("success");
    } else {
        printf("failed");
    }

    return 0;
}

程序反编译

上面的代码编译后，会生成一个可执行程序。逆向分析的第一步，就是把可执行文件放进反编译工具里，观察它还原出的逻辑结构。

反编译结果不一定和源码完全一致，但关键控制流通常可以看出来：输入、比较、分支、输出。

可执行文件格式

Windows 常见的是 PE 文件，Linux 常见的是 ELF 文件。学习逆向时，不需要一开始就背完整格式，先能定位字符串、入口函数和关键调用即可。

CPU 指令集

从源码到机器码，中间会经过编译器优化。理解寄存器、栈、函数调用约定，可以帮助我们把汇编片段重新对应到高级语言逻辑。

比较逻辑

像 strcmp 这样的函数，通常会在反编译结果里直接暴露出字符串常量。真实程序会更复杂，可能会加密字符串、拆分比较或混淆控制流。

高级语言特性

C、C++、Go、Rust 等语言编译后的痕迹不同。学习时建议先从 C 语言开始，因为它的运行时和抽象层更少，反编译结果更接近底层行为。

软件调试

静态分析看结构，动态调试看运行。断点、单步、内存窗口和寄存器窗口，是验证猜想最直接的方式。

结语

逆向分析不是直接背工具按钮，而是训练一种还原程序意图的能力：先找输入和输出，再找判断条件，最后再补齐数据流和控制流。
